# Стиль кода

## Циклы и итерации

- Не использовать классические циклы `for`, только в самых редких случаях
- Использовать методы массивов: `forEach`, `map`, `filter`, `reduce`, `find`, `some`, `every`

## Типы

- Определяй типы через глобальные ambient-декларации в .d.ts, чтобы они были доступны во всем проекте без импортов
- Никогда не объявлять типы в середине кода
- Для проверки optional полей (особенно boolean) использовать `data.key !== undefined`

## Организация кода

- Переиспользуемые функции выносить в `utils/` или `helpers`
- Если какой-то код повторяется более 2 раз - нужно вынести его в отдельную функцию

- Все сообщения об ошибках и `console.log` писать на русском языке
- Не ставить точку в конце предложения, если после него не идёт следующее

## Архитектура

- Не изобретать лишний функционал — делать только то, что просят
